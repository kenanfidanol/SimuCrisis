SimuCrisis – Project Specifications Report

1. Introduction

In today’s world, the ability to manage health crises is very important. Pandemics affect many parts of life such as health, economy, and social structure. SimuCrisis is an educational simulation that helps people understand how different decisions during a pandemic can change the results. SimuCrisis is developed with Unity. SimuCrisis uses data and simple disease models to show how viruses spread and how prevention policies about pandemics work. Through the system, users can test prevention policies like lockdowns, vaccination plans or increasing hospital capacity and see how these prevention policies affect infection rates and healthcare systems.

On the initial plan of the project, the development team included Virtual Reality support to provide a more engaging experience to users. After further evaluation, the development team decided to remove the VR component since the VR environment did not provide meaningful improvements to the project’s ultimate goals which are enhanced educational functions and usability. While it offered visual novelty, it did not change the educational or analytical value of the system and could have complicated usage for users without VR hardware. Therefore, SimuCrisis now focuses solely on the Unity-based desktop version, which provides the same level of interaction and accessibility in a more practical form.

SimuCrisis gives users an interactive and visual way to explore crisis management. The primary goal of SimuCrisis is to ease the learning process about pandemics for educators, researchers and policymakers. The project combines specific models with an interface to increase awareness and better understanding. SimuCrisis provides graphs, heatmaps and reports to users to provide better observation.

1.1 Description

SimuCrisis is a digital simulation tool that lets users control and observe a virtual population during a pandemic. Users can change values like infection rate, recovery time, or medical capacity, and see the results instantly. They can apply actions such as closing schools, setting restrictions, or starting vaccination programs. The program then calculates the new situation and shows the effects visually in real time.

SimuCrisis’ purpose is to help users to understand how prevention strategies work about pandemics and simulate possible challenges decision-makers face. SimuCrisis focuses on being simple, flexible and informative. Hence, it can be used in both classrooms and research studies. The removal of VR ensures that SimuCrisis is easy to use for everyone. The removal of VR increases the hardware availability and its educational effectiveness.

1.2 Constraints

The project must follow several important limits such as economic, environmental, social, political, ethical, health, manufacturability, and sustainability factors. These factors which are listed below make the system more realistic, environment-friendly and durable in the long term.

Economic Constraints: The project is developed with limited time and resources. Unity’s free version is used to avoid licensing costs. Free or open-source data and tools are preferred to keep the budget low. The team also focuses on performance optimization to make sure the program runs smoothly on normal computers.

Environmental Constraints: Although SimuCrisis does not affect the environment directly, energy consumption is minimized. SimuCrisis is designed to use local data processing instead of heavy cloud services to reduce energy consumption.

Social Constraints: Pandemics is a sensitive topic. Hence, SimuCrisis will avoid spreading misinformation. The project will use neutral, educational examples. SimuCrisis includes accessible and friendly design such as readable fonts, clear colors, and simple navigation.

Political Constraints: Every country handles pandemics differently, so SimuCrisis does not copy or promote any real country’s policies. All situations are fictional and created for learning only.

Ethical Constraints: The simulation is not a real-world prediction tool. It only helps users understand how pandemics behave under simple models. All data used is public or artificial. No personal or private health data is collected or shown.

Health and Safety Constraints: SimuCrisis is not a medical or diagnostic program. It only presents educational visualizations about virus spread and recovery. It includes clear warnings so users know it is not for professional use.

Manufacturability Constraints: To serve the purpose of adding new features and data to the system, SimuCrisis is designed with a modular structure. The initial release of SimuCrisis is focused on desktop computers.

Sustainability Constraints: The main aspects of the program such as source code and data handling will follow clean and maintainable design. The SimuCrisis is built to include new features in the future and can be reused for similar crisis simulations.

1.3 Professional and Ethical Issues

The SimuCrisis project and its development team will remain honest, fair and responsible in terms of data usage and system design.

Accuracy and Transparency: All of the data sources and assumptions will be clarified in the documentation. The purpose of clarification is to show calculation of results and limit boundaries.

Fairness and Neutrality: The project will not show any cultural or political bias. All simulations stay general and balanced, only for learning purposes.

Privacy and Data Use: Shared data from universal resources such as the World Health Organization will be used. SimuCrisis will not process or store any personal information.

Professional Responsibility: The development team will stay honest about the capabilities of the system and limits. The development team will ensure that simulation works as described and does not mislead users.

Public Welfare: Since the project includes health-related content, it clearly states that it is only an educational tool and not a replacement for medical or professional advice.

2. Requirements

Functional Requirements

  • The system should simulate virus spread based on infection, recovery, and contact rates.
  • It should let users apply strategies like lockdowns, vaccination, and healthcare investments.
  • It should display real-time graphs showing infections, deaths, and hospital usage.
  • It should allow users to compare different policy results.
  • It should include example data from past pandemics such as COVID-19 and H1N1.

Non-Functional Requirements

  • Usability: The interface should be simple and clear for teachers and students.
  • Performance: The program should run smoothly and respond quickly.
  • Maintainability: The code should be modular and easy to update.
  • Scalability: The system should allow adding new datasets and features.
  • Reliability: Results should be correct and consistent each time the simulation runs.
