1. Introduction

In today’s world, the ability to manage health crises is very important. Pandemics affect many parts of life such as health, economy, and social structure. SimuCrisis is an educational simulation that helps people understand how different decisions during a pandemic can change the results. The project is developed with Unity and uses data and simple disease models to show how viruses spread and how management decisions work. Through this system, users can test ideas like lockdowns, vaccination plans, or increasing hospital capacity and see how these decisions affect infection rates and healthcare systems.

SimuCrisis gives users an interactive and visual way to explore crisis management. It provides graphs, heatmaps, and reports to show how the situation changes over time. The goal is to make learning about pandemics easier for educators, researchers, and policymakers. The project combines scientific models with an easy-to-use interface to support better understanding and awareness.

1.1 Description

SimuCrisis is a digital simulation tool that lets users control and observe a virtual population during a pandemic. Users can change values like infection rate, recovery time, or medical capacity, and see the results instantly. They can apply actions such as closing schools, setting restrictions, or starting vaccination programs. The program then calculates the new situation and shows the effects visually in real time.

The purpose of SimuCrisis is to help users understand how different strategies work and what challenges decision-makers face. It focuses on being simple, flexible, and informative so that it can be used in both classrooms and research studies.

1.2 Constraints

The project must follow several important limits such as economic, environmental, social, political, ethical, health, manufacturability, and sustainability factors. These help make sure that the system is realistic, responsible, and long-lasting.

Economic Constraints: The project is developed with limited time and resources. Unity’s free version is used to avoid licensing costs. Free or open-source data and tools are preferred to keep the budget low. The team also focuses on performance optimization to make sure the program runs smoothly on normal computers.

Environmental Constraints: Even though the system does not affect the environment directly, energy use is minimized. SimuCrisis is designed to use local data processing instead of heavy cloud services to reduce energy consumption.

Social Constraints: Because the topic of pandemics can be sensitive, the project avoids spreading misinformation or fear. It uses neutral, educational examples and includes accessibility-friendly design such as readable fonts, clear colors, and simple navigation.

Political Constraints: Every country handles pandemics differently, so SimuCrisis does not copy or promote any real country’s policies. All situations are fictional and created for learning only.

Ethical Constraints: The simulation is not a real-world prediction tool. It only helps users understand how pandemics behave under simple models. All data used is public or artificial. No personal or private health data is collected or shown.

Health and Safety Constraints: SimuCrisis is not a medical or diagnostic program. It only presents educational visualizations about virus spread and recovery. It includes clear warnings so users know it is not for professional use.

Manufacturability Constraints: The program is made with a modular structure so new data and features can be added easily. The first release is focused on desktop computers, but a web version may be developed later.

Sustainability Constraints: The code and data handling follow clean, maintainable design rules. The system is built to support future improvements and can be reused for similar crisis simulations such as earthquakes or climate disasters.

1.3 Professional and Ethical Issues

SimuCrisis follows professional and ethical principles from the ACM and IEEE codes. It focuses on honesty, fairness, and responsibility in both data use and system design.

Accuracy and Transparency: All formulas, data sources, and assumptions will be clearly explained in the documentation. This helps users understand how results are calculated and where the limits are.

Fairness and Neutrality: The project will not show any cultural or political bias. All simulations stay general and balanced, only for learning purposes.

Privacy and Data Use: Only public or artificial data will be used. The system does not store or process any personal information.

Professional Responsibility: The team will stay honest about the system’s abilities and limits. They will make sure the simulation works as described and does not mislead users.

Public Welfare: Since the project includes health-related content, it clearly states that it is only an educational tool and not a replacement for medical or professional advice.

2. Requirements

Functional Requirements

  • The system should simulate virus spread based on infection, recovery, and contact rates.
  • It should let users apply strategies like lockdowns, vaccination, and healthcare investments.
  • It should display real-time graphs showing infections, deaths, and hospital usage.
  • It should allow users to compare different policy results.
  • It should include example data from past pandemics such as COVID-19 and H1N1.

Non-Functional Requirements

  • Usability: The interface should be simple and clear for teachers and students.
  • Performance: The program should run smoothly and respond quickly.
  • Maintainability: The code should be modular and easy to update.
  • Scalability: The system should allow adding new datasets and features.
  • Reliability: Results should be correct and consistent each time the simulation runs.
