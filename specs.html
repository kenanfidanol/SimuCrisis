SimuCrisis – Project Specifications Report

1. Introduction

In today’s world, the ability to manage health crises is very important. Pandemics affect many parts of life such as health, economy, and social structure. SimuCrisis is an educational simulation that helps people understand how different decisions during a pandemic can change the results. The project is developed with Unity and uses data and simple disease models to show how viruses spread and how management decisions work. Through this system, users can test ideas like lockdowns, vaccination plans, or increasing hospital capacity and see how these decisions affect infection rates and healthcare systems.

SimuCrisis gives users an interactive and visual way to explore crisis management. It provides graphs, heatmaps, and reports to show how the situation changes over time. The goal is to make learning about pandemics easier for educators, researchers, and policymakers. The project combines scientific models with an easy-to-use interface to support better understanding and awareness.

1.1 Description

SimuCrisis is a digital simulation tool that lets users control and observe a virtual population during a pandemic. Users can change values like infection rate, recovery time, or medical capacity, and see the results instantly. They can apply actions such as closing schools, setting restrictions, or starting vaccination programs. The program then calculates the new situation and shows the effects visually in real time.

The purpose of SimuCrisis is to help users understand how different strategies work and what challenges decision-makers face. It focuses on being simple, flexible, and informative so that it can be used in both classrooms and research studies.

1.2 Constraints

The project must follow several important limits such as economic, environmental, social, political, ethical, health, manufacturability, and sustainability factors. These help make sure that the system is realistic, responsible, and long-lasting.

Economic Constraints: Developed with limited resources using Unity’s free version and open-source tools to minimize cost.

Environmental Constraints: Optimized for local data processing to reduce energy use instead of relying on cloud computing.

Social Constraints: Avoids misinformation and fear; focuses on educational and accessible design.

Political Constraints: Uses fictional, neutral scenarios for learning purposes only.

Ethical Constraints: Does not predict real-world outcomes or use personal data; only public/artificial datasets.

Health & Safety Constraints: Not a diagnostic or medical tool; purely educational.

Manufacturability Constraints: Modular structure allows future updates and web conversion.

Sustainability Constraints: Built for reusability and future expansion for other crisis simulations.

1.3 Professional and Ethical Issues

SimuCrisis follows professional and ethical principles from ACM and IEEE codes, emphasizing honesty, fairness, and responsibility.

Accuracy and Transparency: Documentation explains all formulas and assumptions clearly.

Fairness and Neutrality: No cultural or political bias; designed purely for learning.

Privacy and Data Use: Only public or artificial data is used, no personal data collection.

Professional Responsibility: Developers ensure the tool functions as described without misleading users.

Public Welfare: The tool states clearly that it is not for medical or professional advice.

2. Requirements

Functional Requirements

  • Simulate virus spread using infection, recovery, and contact rates.
  • Allow users to apply strategies like lockdowns, vaccination, and healthcare investments.
  • Display real-time graphs for infections, deaths, and hospital use.
  • Enable users to compare different policy results.
  • Include example datasets from COVID-19 and H1N1.

Non-Functional Requirements

  • Usability: Clear, user-friendly interface for educators and students.
  • Performance: Fast and responsive simulation.
  • Maintainability: Modular and easy-to-update code.
  • Scalability: Supports adding new data and features.
  • Reliability: Produces consistent and correct results each run.
