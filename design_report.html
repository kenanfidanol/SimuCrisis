High Level Design Report

SimuCrisis – Pandemic Disaster Management Simulation

1. Introduction

1.1 Purpose of the System

SimuCrisis is a pandemic simulation system developed using Unity for educational and decision-support purposes. The system enables users to explore how pandemic management decisions—such as lockdowns, vaccination strategies, and healthcare capacity adjustments— affect virus spread and healthcare system load.

The system targets policymakers, educators, researchers, and students who wish to gain insight into pandemic dynamics in a controlled, risk-free environment.

1.2 Design Goals

  • Clear and intuitive visualization of pandemic dynamics
  • Compatibility with standard desktop hardware (VR excluded)
  • Real-time feedback for interventions
  • Modular, maintainable, and scalable architecture
  • Reproducibility using saved scenarios and random seeds

1.3 Definitions and Abbreviations

  • SEIR: Susceptible, Exposed, Infected, Recovered model
  • Rt: Effective reproduction number
  • Scenario: Stored simulation configuration
  • Intervention: User-applied policy (lockdown, vaccination, etc.)

2. Proposed Software Architecture

SimuCrisis follows a modular, layered architecture composed of three main layers: User Interface Layer, Core Layer, and Data Layer. This separation ensures maintainability, testability, and future extensibility.

3. Subsystem Services

Simulation Core

Executes the SEIR epidemiological model and updates population states at each timestep. It validates parameters and prevents unstable simulation states.

Policy Engine

Processes user-defined interventions and modifies simulation parameters dynamically. Validates policy constraints to avoid logical conflicts.

Metrics Collector

Computes analytical indicators such as Rt, infection peaks, and hospital utilization. Provides time-series data for visualization and reporting.

Visualization Engine

Renders charts and spatial heatmaps that visualize infection spread and trends in real time. Optimized to maintain performance on standard hardware.

Scenario Manager

Handles saving, loading, and comparison of simulation scenarios using structured JSON files.

Report Generator

Exports simulation results into PDF summaries and CSV datasets for external analysis.

4. Runtime Interaction

The system operates in an event-driven loop. User actions initiate simulation runs, policy applications, and parameter updates, which propagate through the system layers.
